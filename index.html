PyRegisterMachine

a register machine written in c and python3

What PyRegisterMachine is about

The goal of PyRegisterMachine is it to provide a generic register machine package.
Using such a register machine package is not like just emulating a real processor, it is designed for scientific and educational purposes and comes with features that could not provided by such an emulation.

Scientific advantages

  • PyRegisterMachine is generic.
    Using a generic engine allows you to select just the mnemonics you need and/or you want.
    In Addition to this PyRegisterMachine makes it really easy to add new mnemonics, if you need them.
  • PyRegisterMachine has special debug features.
    PyRegistermachine automatically logs the number of operations and saves the complete call stack.
    This allows to generate data about algorithm run-times and is suitable to test algorithms.
  • PyRegisterMachine allows dumps.
    Reading the content of your RAM or Flash is important, if your are developing on machine level or debugging algorithms.
    PyRegisterMachine has the mnemonic fdump for dumping the flash to a file and a SpecialFunctionRegister command to dump the RAM to a file.

  • PyRegisterMachine has IORegister s.
    Those Registers allow the register machine to

    • Communicate with the environment

    And allows an user to

    • Implement Virtual Discs
    • Implement Internet Access
  • PyRegisterMachine supports interrupts.
    Interrupts are essential for the development of Operating Systems.
    Including the other features of the engine, PyRegisterMachine is perfect for developing Operating Systems, as scientific or educational project.
  • KASM the Assembly Language for PyRegisterMachine is easy to learn and easy to process.
    This allows
    • New users to jump quickly into programming whith KASM
    • Developing new assemblers easily
    • Developing High-Level languages based on KASM
